Yönetici Özeti

FÜ Turnike Yönetim Sistemi, Fırat Üniversitesi'nin kampüs erişim kontrol altyapısını modern bir web arayüzü üzerinden yönetmesine olanak tanıyan, mission control hissiyatında tasarlanmış kapsamlı bir SaaS yönetim panelidir.

Sistem; 49.390 kullanıcı (43.463 öğrenci + 5.927 personel), 46.625 aktif kart ve 6 turnike cihazını tek ekrandan izleyebilmeyi, gerçek zamanlı geçiş loglarını takip etmeyi ve istatistiksel analizler yapabilmeyi mümkün kılmaktadır.

Mimari Kararlar

Neden Vite? Create React App'e kıyasla 10-20x daha hızlı cold start ve HMR süresi sağlar. Büyük ölçekli projeler için kritik avantaj.
Neden Framer Motion? React ekosistemindeki en kapsamlı animasyon kütüphanesi. AnimatePresence ile mount/unmount animasyonları, layout geçişleri mümkün.
Neden Tailwind? Design token sistemi sayesinde renk paleti ve spacing tutarlılığı. Custom CSS ile hibrit kullanım esnek tasarım imkânı sağlar.

Animasyon Sistemi

Tüm animasyonlar iki katmanda uygulanmıştır: CSS keyframe animasyonlar (saf performans için) ve Framer Motion (React state değişimlerine bağlı animasyonlar için).

🔴
Canlı Nabız (Live Pulse)
Topbar ve log başlığındaki yeşil nokta, opacity + scale ile 1.4 saniye döngüde pulse atar. İki katmanlı: iç nokta (solid) + dış halka (fade-out ring). CSS keyframe: livePulse + livePulseRing.
📈
Count-Up Sayaçlar
Hem useCountUp hook'u (sayfa açılışı için) hem de AnimatedNumber bileşeni (canlı güncelleme için) kullanılmıştır. Ease-out cubic eğrisi ile 400ms'de tween yapılır. Her artışta sayı kısa süre mor (#a78bfa) renk alır.
📋
Log Satırı Slide-In
Framer Motion AnimatePresence ile yönetilir. Yeni satır üstten y: -20, opacity: 0'dan y: 0, opacity: 1'e 350ms'de gelir. Arka plan başlangıçta rgba(124,58,237,0.15) ile flash yapar, sonra şeffafa döner.
📊
Bar Chart Açılma
Her bar scaleY: 0 → 1 ile yukarı açılır. transformOrigin: bottom. Staggered delay: her bar öncekinden 70ms sonra başlar. Ease: [0.22, 1, 0.36, 1] (spring benzeri).
📉
Progress Bar Dolma
Framer Motion animate={{ width: percent% }} ile duration: 0.5s, ease: easeOut. Canlı simülasyon aktifken her geçişte ilgili turnike progress bar'ı ±1 değişir, yüzde sayısı scale: 1.3 → 1 ile bounce atar.
🃏
Kart Hover Efekti
CSS transition: transform: translateY(-3px) + border rengi rgba(124,58,237,0.35)'e geçiş. 200ms ease. Devices sayfasında Framer Motion whileHover ile yönetilir.
🪟
Modal Açılma/Kapanma
Devices sayfası detay modal'ı: arka plan opacity: 0→1, kart scale: 0.92, y: 20 → scale: 1, y: 0. Spring animasyon: damping: 20, stiffness: 300. Dışına tıklayınca exit animasyonu ile kapanır.
🔄
Yenile Butonu Spin
Topbar'daki "Yenile" butonu tıklandığında ikon animate-spin ile 1 saniye döner, ardından durur. State: refreshing: true → false.

Canlı Simülasyon Mimarisi

Gerçek WebSocket bağlantısı olmadan sistemin canlı davranışını test edebilmek için useSimulation hook'u geliştirilmiştir. Hook, her 3 saniyede bir rastgele bir geçiş olayı üretir ve tüm UI'ı günceller.

useSimulation Hook'u — Veri Akışı

setInterval (3000ms)
→
generateEntry()
→
Rastgele kullanıcı + cihaz + saat
↓
setLogs → Log tablosu güncellenir
setStats → todayAccess/successfulAccess/failedAccess +1
setTurnikeData → İlgili cihazın yoğunluğu ±1

Gerçek Backend ile Değiştirme

Simülasyondan WebSocket'e geçiş yalnızca useSimulation.js dosyasındaki setInterval bloğunun yerine aşağıdaki kodun yazılmasıyla yapılır — bileşenlerde hiçbir değişiklik gerekmez. 
// useSimulation.js — WebSocket versiyonu
const socket = new WebSocket('ws://api.firat.edu.tr/ws/access')

socket.onmessage = (event) => {
  const entry = JSON.parse(event.data)
  // entry: { name, type, device, time, status, initials, ... }

  setLogs(prev => [entry, ...prev.slice(0, 7)])

  setStats(prev => ({
    ...prev,
    todayAccess:      prev.todayAccess + 1,
    totalAccess:      prev.totalAccess + 1,
    successfulAccess: entry.status === 'BAŞARILI'
      ? prev.successfulAccess + 1 : prev.successfulAccess,
    failedAccess: entry.status === 'BAŞARISIZ'
      ? prev.failedAccess + 1 : prev.failedAccess,
  }))
}

Önerilen Veritabanı Şeması (PostgreSQL)

-- Kullanıcılar
CREATE TABLE users (
  id       SERIAL PRIMARY KEY,
  name     VARCHAR(100),
  type     VARCHAR(30),   -- 'Öğrenci' | 'Çalışan' | 'Öğretim Görevlisi'
  dept     VARCHAR(100),
  active   BOOLEAN DEFAULT TRUE
);

-- Kartlar
CREATE TABLE cards (
  id       SERIAL PRIMARY KEY,
  card_no  VARCHAR(20) UNIQUE,
  user_id  INTEGER REFERENCES users(id),
  status   VARCHAR(10) DEFAULT 'Aktif'
);

-- Geçiş logları
CREATE TABLE access_logs (
  id        SERIAL PRIMARY KEY,
  card_id   INTEGER REFERENCES cards(id),
  device_id VARCHAR(10),
  status    VARCHAR(15),  -- 'BAŞARILI' | 'BAŞARISIZ'
  ts        TIMESTAMP DEFAULT NOW()
);
